1. Difference between map, filter, forEach and find .

In JavaScript, map, filter, forEach, and find are all methods used to manipulate arrays.

Map: The map method creates a new array with the result of calling a given function on each element of the original array. It does not change the original array.

Filter: The filter method creates a new array with all elements that pass the test implemented by the given function. It does not change the original array.

forEach: The forEach method executes the given function once for each array element. It doesn't create a new array and doesn't return anything.

find: The find method returns the value of the first element of the array that satisfies the given function. It does not create a new array.

2. Difference between Local Storage, Session Storage and Cookies .

In web development, local storage, session storage, and cookies are all mechanisms for storing data on the client-side.

localStorage: The LocalStorage API allows you to store key-value pairs in the browser's local storage. Data persists even when the user closes their browser or restarts their computer. localStorage is specific to each domain, meaning that data stored in local storage by one website cannot be accessed by another website.

sessionStorage: The SessionStorage API is similar to LocalStorage, but data is only stored for the duration of the session. Data is lost when the user closes their browser or tab. Session storage is specific to each domain.

cookies: Cookies are small text files that are stored on the user's computer by the browser. Cookies may be used to store data such as user preferences, login information, and shopping cart contents. Unlike local storage and session storage, cookies are sent to the server with each request, allowing the server to read the data.

3. What is the event loop? how does it work?

JavaScript has a runtime model based on an event loop, which is responsible for executing the code, collecting and processing events, and executing queued sub-tasks. This model is quite different from models in other languages like C and Java.

The event loop is a process that waits for the Call Stack to be clear before pushing callbacks from the Task Queue to the Call Stack. Once the Stack is clear, the event loop triggers and checks the Task Queue for available callbacks. If there are any, it pushes it to the Call Stack, waits for the Call Stack to be clear again, and repeats the same process.

This function chooseQuiz accepts a quiz number and a replacement answer. Then this function finds an array identifier (id) from the given quiz number and the given answer to see if it is in the array or not. If it is, replaces the old answer with the new answer and updates it to the array. If it doesn't exist on the contrary, it creates a new object and adds the new quiz and the given answer to the array. Finally, after the function completes, another function returns the array with the answers to displayAnswers to display to the user.
